## Active Issues

1. **CSV Import Failure**: 
   - Importing from CSV clears the board but fails to populate the list with URLs. 
   - **Root Cause**: The CSV parsing logic implementation was possibly omitted or is buggy in `popup.js`.

2. **Auto-Collection Logic (YouTube Shorts)**:
   - **Scenario**: When scrolling YouTube Shorts, the URL updates in the address bar (History API pushState), but `chrome.tabs.onUpdated` might not be triggering 'complete' status in the way we expect, or we are catching the "previous" URL before the new one settles.
   - **Specific Symptom**: It "continues to get the previous short url" or misses the new one unless explicitly moved further.
   - **Diagnosis**: YouTube relies heavily on `pushState` for navigation without full page reloads. Our listener checks `changeInfo.status === 'complete'`, which happens on load, but for SPA navigation (Shorts), we need to listen for **URL changes specifically**.

## Next Steps
- Implement robust CSV parsing logic.
- Refine `background.js` listener to handle `changeInfo.url` explicitly for Single Page Applications (SPAs) like YouTube, rather than waiting for `status: complete`.
